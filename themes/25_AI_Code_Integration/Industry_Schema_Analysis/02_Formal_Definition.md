## 1. 行业分类

### 1.1 按标准化程度分类

- **高度标准化**：金融服务、医疗健康、物流供应链
- **中等标准化**：工业自动化、智慧城市、交通
- **低标准化**：其他行业、新兴行业

### 1.2 按数据格式分类

- **文本格式**：EDI、HL7、SWIFT
- **JSON格式**：OpenAPI、AsyncAPI、FHIR
- **XML格式**：EDIFACT、BPMN、SCL
- **二进制格式**：IoT协议、工业协议

---

## 2. 标准对比

| 行业 | 主要标准 | 标准化程度 | 数据格式 |
|------|---------|-----------|---------|
| **金融服务** | ISO20022、SWIFT | 高 | XML、文本 |
| **医疗健康** | HL7、FHIR | 高 | XML、JSON |
| **物流供应链** | GS1、EDI | 高 | XML、文本 |
| **工业自动化** | IEC 61131-3、ISO 11783 | 中 | 二进制、XML |
| **智慧城市** | ISO 14813、IEEE 1609 | 中 | JSON、XML |

---

## 3. 数据格式对比

### 3.1 文本格式

- **优势**：人类可读、易于调试
- **劣势**：体积大、解析慢

### 3.2 JSON格式

- **优势**：轻量级、易于解析
- **劣势**：不支持注释、类型信息有限

### 3.3 XML格式

- **优势**：结构化、支持验证
- **劣势**：体积大、解析复杂

---

## 4. 转换复杂度对比

| 转换类型 | 复杂度 | 原因 |
|---------|--------|------|
| **EDI到GS1** | 中 | 语义相似，格式不同 |
| **HL7到FHIR** | 低 | 同行业，标准演进 |
| **ISO20022到SWIFT** | 中 | 同行业，格式不同 |
| **IoT到OpenAPI** | 高 | 协议差异大 |
