## 1. 核心结论

**保险业务存在强制性的Schema体系，涵盖寿险、财险、健康险、再保险等多个业务领域**。

### 1.1 保险业务Schema定义

```text
Insurance_Schema = (Life_Insurance ⊕ Property_Insurance ⊕ Health_Insurance ⊕ Reinsurance) × Regulatory_Profile
```

其中：

- `Life_Insurance`：人寿保险Schema
- `Property_Insurance`：财产保险Schema
- `Health_Insurance`：健康保险Schema
- `Reinsurance`：再保险Schema
- `Regulatory_Profile`：监管合规配置

### 1.2 标准依据

- **ACORD**：保险数据交换标准
- **Solvency II**：偿付能力监管标准
- **IFRS 17**：保险合同会计标准
- **GDPR/数据保护法规**：客户数据保护标准

## 2. 概念定义

### 2.1 保险业务Schema

**保险业务Schema**是描述保险业务数据的形式化规范，包括保单信息、投保人信息、被保险人信息、受益人信息、理赔信息、保费信息等核心业务数据实体。

### 2.2 核心特征

1. **长期性**：保险合同通常具有长期性特征
2. **复杂性**：保险条款和责任复杂多样
3. **精算性**：依赖精算模型进行定价和准备金计算
4. **合规性**：受多重监管要求约束
5. **服务性**：保险本质是一种金融服务

### 2.3 Schema分类

- **寿险Schema**：定期寿险、终身寿险、年金保险、投资连结保险
- **财险Schema**：车险、企财险、家财险、责任险、货运险
- **健康险Schema**：医疗费用保险、疾病保险、失能收入保险、护理保险
- **再保险Schema**：比例再保险、非比例再保险

## 3. 保险核心业务Schema

### 3.1 人寿保险Schema

**定义**：描述人寿保险保单和相关业务的数据结构。

**包含内容**：

- **定期寿险**：固定期限、纯保障型
- **终身寿险**：终身保障、储蓄型
- **年金保险**：养老金、教育金、生存年金
- **投资连结保险**：投连险、万能险、分红险

**核心属性**：

| 属性 | 类型 | 说明 | 约束 |
|------|------|------|------|
| policy_number | String(30) | 保单号 | 唯一，必填 |
| product_code | String(20) | 产品代码 | 必填 |
| policy_type | Enum | 保单类型 | TERM, WHOLE_LIFE, ANNUITY, UL |
| sum_assured | Decimal(15,2) | 保险金额 | > 0 |
| premium | Decimal(15,2) | 保费 | > 0 |
| premium_frequency | Enum | 缴费频率 | YEARLY, HALF_YEARLY, QUARTERLY, MONTHLY |
| policy_term | Integer | 保险期间(年) | > 0 |
| premium_term | Integer | 缴费期间(年) | > 0 |
| policy_status | Enum | 保单状态 | INFORCE, LAPSED, PAID_UP, MATURED |

**寿险产品对比**：

| 产品类型 | 保障期限 | 储蓄功能 | 投资功能 | 适合人群 |
|----------|----------|----------|----------|----------|
| 定期寿险 | 固定期限 | 无 | 无 | 家庭经济支柱 |
| 终身寿险 | 终身 | 有 | 无 | 财富传承 |
| 年金保险 | 终身/定期 | 有 | 可选 | 养老规划 |
| 投连险 | 终身 | 有 | 有 | 风险承受能力高 |
| 万能险 | 终身 | 有 | 有 | 灵活规划 |
| 分红险 | 终身/定期 | 有 | 有 | 稳健型投资者 |

### 3.2 财产保险Schema

**定义**：描述财产保险保单和相关业务的数据结构。

**包含内容**：

- **机动车辆保险**：交强险、商业车险（车损险、三者险、盗抢险等）
- **企业财产保险**：财产损失险、营业中断险
- **家庭财产保险**：房屋保险、家财综合险
- **责任保险**：公众责任险、雇主责任险、职业责任险
- **货物运输保险**：国内货运险、进出口货运险

**车险核心属性**：

| 属性 | 类型 | 说明 | 约束 |
|------|------|------|------|
| policy_number | String(30) | 保单号 | 唯一，必填 |
| vehicle_license | String(20) | 车牌号 | 必填 |
| vehicle_type | Enum | 车辆类型 | PRIVATE, COMMERCIAL |
| use_nature | Enum | 使用性质 | NON_PROFIT, PROFIT |
| vehicle_value | Decimal(12,2) | 车辆价值 | > 0 |
| coverage_types | List | 投保险种 | 必填 |
| premium | Decimal(12,2) | 保费 | > 0 |
| effective_date | Date | 生效日期 | 必填 |
| expiry_date | Date | 到期日期 | 必填 |
| no_claim_discount | Decimal(3,2) | 无赔款优待 | 0-1 |

**商业车险险种**：

|----------|----------|----------|----------|

| B | 第三者责任保险 | 第三方人身财产损失 | 主险 |
| D | 车上人员责任险 | 车上人员伤亡 | 主险 |
| G | 全车盗抢保险 | 整车被盗抢 | 主险 |
| Z1 | 玻璃单独破碎险 | 玻璃单独损坏 | 附加 |
| Z2 | 自燃损失险 | 车辆自燃损失 | 附加 |
| Z3 | 新增加设备损失险 | 新增设备损失 | 附加 |
| Z4 | 车身划痕损失险 | 车身划痕 | 附加 |
| Z5 | 发动机涉水损失险 | 涉水损坏 | 附加 |
| Z6 | 修理期间费用补偿险 | 代步费用 | 附加 |
| Z7 | 车上货物责任险 | 车上货物损失 | 附加 |
| Z8 | 精神损害抚慰金责任险 | 精神损害赔偿 | 附加 |
| Z9 | 不计免赔率险 | 免赔额补偿 | 附加 |
| Z10 | 机动车损失保险无法找到第三方特约险 | 找不到第三方 | 附加 |

### 3.3 健康保险Schema

**定义**：描述健康保险保单和相关业务的数据结构。

**包含内容**：

- **医疗费用保险**：住院医疗、门诊医疗、百万医疗
- **疾病保险**：重大疾病保险、特定疾病保险
- **失能收入损失保险**：失能收入补偿
- **护理保险**：长期护理、短期护理

**重疾险核心属性**：

| 属性 | 类型 | 说明 | 约束 |
|------|------|------|------|
| policy_number | String(30) | 保单号 | 唯一，必填 |
| product_code | String(20) | 产品代码 | 必填 |
| sum_assured | Decimal(15,2) | 保额 | > 0 |
| premium | Decimal(12,2) | 保费 | > 0 |
| covered_diseases | List | 保障疾病种类 | 必填 |
| disease_count | Integer | 疾病数量 | > 0 |
| waiting_period | Integer | 等待期(天) | ≥ 0 |
| survival_period | Integer | 生存期(天) | ≥ 0 |

**重疾险疾病分类**：

| 疾病类别 | 数量 | 示例疾病 |
|----------|------|----------|
| 恶性肿瘤 | 1类 | 癌症 |
| 急性心肌梗塞 | 1类 | - |
| 脑中风后遗症 | 1类 | - |
| 重大器官移植 | 1类 | 心脏、肝脏、肾脏移植 |
| 冠状动脉搭桥术 | 1类 | - |
| 终末期肾病 | 1类 | 尿毒症 |
| 其他重疾 | 25-100+类 | 多个肢体缺失、良性脑肿瘤等 |
| 中症疾病 | 20-30类 | 中度脑中风、中度类风湿等 |
| 轻症疾病 | 30-50类 | 极早期恶性肿瘤、轻微脑中风等 |

### 3.4 再保险Schema

**定义**：描述再保险合约和分保业务的数据结构。

**包含内容**：

- **比例再保险**：成数再保险、溢额再保险
- **非比例再保险**：超赔再保险、 stop loss再保险

**再保险核心属性**：

| 属性 | 类型 | 说明 | 约束 |
|------|------|------|------|
| treaty_id | String(30) | 合约号 | 唯一，必填 |
| treaty_type | Enum | 合约类型 | QUOTA_SHARE, SURPLUS, XL, STOP_LOSS |
| cedant | String(50) | 分出公司 | 必填 |
| reinsurer | String(50) | 接受公司 | 必填 |
| line_of_business | String(50) | 业务线 | 必填 |
| cession_rate | Decimal(5,2) | 分保比例 | 0-1 |
| retention_limit | Decimal(15,2) | 自留额 | ≥ 0 |
| treaty_limit | Decimal(15,2) | 合约限额 | > 0 |
| effective_date | Date | 生效日期 | 必填 |
| expiry_date | Date | 到期日期 | 必填 |

**再保险类型对比**：

| 类型 | 保费分保 | 赔款分保 | 适用场景 |
|------|----------|----------|----------|
| 成数再保险 | 固定比例 | 固定比例 | 新业务、小型公司 |
| 溢额再保险 | 按溢额比例 | 按溢额比例 | 业务稳定、大型公司 |
| 超赔再保险 | 不比例 | 超限额部分 | 巨灾风险 |
| stop loss | 不比例 | 年度累计超赔 | 年度风险管控 |

### 3.5 保险资产管理Schema

**定义**：描述保险资金投资和资产管理业务的数据结构。

**包含内容**：

- **投资组合**：股票、债券、基金、另类投资
- **资产负债管理**：久期匹配、现金流匹配
- **风险管理**：市场风险、信用风险、流动性风险

**投资资产属性**：

| 属性 | 类型 | 说明 | 约束 |
|------|------|------|------|
| asset_id | String(30) | 资产ID | 唯一，必填 |
| asset_type | Enum | 资产类型 | EQUITY, BOND, FUND, ALTERNATIVE |
| asset_name | String(200) | 资产名称 | 必填 |
| isin | String(12) | ISIN编码 | 可选 |
| purchase_price | Decimal(15,4) | 买入价格 | > 0 |
| quantity | Decimal(15,4) | 持有数量 | > 0 |
| market_value | Decimal(15,2) | 市值 | > 0 |
| purchase_date | Date | 买入日期 | 必填 |
| maturity_date | Date? | 到期日期 | 可选 |
| credit_rating | String(10)? | 信用评级 | 可选 |

## 4. 标准对标

### 4.1 ACORD标准

**标准名称**：Association for Cooperative Operations Research and Development

**核心内容**：

- **PC-ACORD**：财产意外险数据标准
- **LA-ACORD**：寿险年金数据标准
- **Re-ACORD**：再保险数据标准
- **Surety-ACORD**：保证险数据标准

**Schema支持**：完整支持

**参考链接**：
[ACORD官网](https://www.acord.org/)

### 4.2 Solvency II

**标准名称**：Solvency II Directive (2009/138/EC)

**核心内容**：

- **三支柱体系**：
  - 第一支柱：定量要求（资本要求、准备金）
  - 第二支柱：治理与风险管理
  - 第三支柱：信息披露与透明度

- **偿付能力资本要求（SCR）**：
  - 标准公式法
  - 内部模型法

- **最低资本要求（MCR）**：
  - SCR的25%-45%

### 4.3 IFRS 17

**标准名称**：IFRS 17 Insurance Contracts

**核心内容**：

- **保险合同确认**：保险合同的识别和分拆
- **计量模型**：
  - 一般模型（GMM）
  - 保费分配法（PAA）
  - 可变费用法（VFA）
- **保险合同负债**：履约现金流、合同服务边际
- **列报与披露**：财务报表列报要求

**生效日期**：2023年1月1日

### 4.4 其他相关标准

| 标准 | 说明 | 适用范围 |
|------|------|----------|
| **GDPR** | 通用数据保护条例 | 欧盟保险业务 |
| **C-ROSS** | 中国风险导向偿付能力体系 | 中国保险业 |
| **SAM** | 南非偿付能力管理 | 南非保险业 |
| **LICAT** | 加拿大保险资本充足率测试 | 加拿大保险业 |

## 5. 应用场景

### 5.1 保单生命周期管理

- **新单承保**：投保申请、核保、保单签发
- **保单保全**：变更、续期、复效、退保
- **保单理赔**：报案、查勘、理算、赔付
- **保单到期**：满期给付、续保

### 5.2 理赔处理系统

- **报案受理**：多渠道报案、案件登记
- **查勘定损**：现场查勘、损失评估
- **理算核赔**：赔款计算、核赔审批
- **赔付结案**：赔款支付、案件结案

### 5.3 精算与风险管理

- **产品定价**：风险费率、费率厘定
- **准备金评估**：未到期责任准备金、未决赔款准备金
- **偿付能力管理**：资本充足率监控
- **资产负债管理**：ALM分析

### 5.4 监管报送

- **偿付能力报告**：季度、年度偿付能力报告
- **财务报告**：IFRS 17财务报告
- **业务统计**：保费收入、赔付支出统计
- **风险报告**：风险评估报告

### 5.5 保险数据存储与分析

**数据库存储应用场景**：

- **PostgreSQL保险数据存储**：
  - 保单信息存储（寿险、财险、健康险）
  - 客户信息存储（投保人、被保险人、受益人）
  - 理赔数据存储（报案、查勘、赔付）
  - 精算数据存储（准备金、赔付率）
  - 再保险数据存储（分保、转分保）

**应用价值**：

- 高效存储大规模保险业务数据
- 支持保险业务分析和报表生成
- 提供保险风险追踪和审计
- 支持保险合规性检查

## 7. 保险业务流程

### 7.1 承保流程

```text
客户咨询 → 需求分析 → 方案设计 → 投保申请 → 核保评估 → 保单签发 → 保费收取 → 保单生效
   │           │           │           │           │           │           │           │
   ▼           ▼           ▼           ▼           ▼           ▼           ▼           ▼
产品介绍     风险评估    产品推荐    填写投保单  健康告知    系统核保    首期保费    保单配送
            需求调研    方案对比    资料收集    财务核保    人工核保    收费确认    生效确认
```

**流程说明**：

1. **客户咨询**：了解客户需求，介绍产品
2. **需求分析**：风险评估，需求调研
3. **方案设计**：产品推荐，方案对比
4. **投保申请**：填写投保单，收集资料
5. **核保评估**：健康告知，财务核保，系统/人工核保
6. **保单签发**：生成保单，保单配送
7. **保费收取**：收取首期保费，收费确认
8. **保单生效**：保单生效，通知客户

### 7.2 理赔流程

```text
客户报案 → 案件登记 → 查勘定损 → 资料收集 → 理算核赔 → 赔款支付 → 案件结案
   │           │           │           │           │           │           │
   ▼           ▼           ▼           ▼           ▼           ▼           ▼
多渠道报案   案件分类    现场查勘    理赔材料    赔款计算    银行转账    案件归档
             任务分配    损失评估    材料审核    核赔审批    支票支付    数据分析
```

**流程说明**：

1. **客户报案**：电话、APP、微信等多渠道报案
2. **案件登记**：案件分类，任务分配
3. **查勘定损**：现场查勘，损失评估
4. **资料收集**：收集理赔材料，材料审核
5. **理算核赔**：赔款计算，核赔审批
6. **赔款支付**：银行转账或支票支付
7. **案件结案**：案件归档，数据分析

### 7.3 保全变更流程

```text
客户申请 → 身份验证 → 变更审核 → 系统处理 → 批单生成 → 通知客户
   │           │           │           │           │           │
   ▼           ▼           ▼           ▼           ▼           ▼
填写申请表   证件核验    合规检查    保单变更    批单打印    短信/邮件
提交材料     人脸识别    风险评估    数据更新    电子批单    通知确认
```

**保全变更类型**：

| 变更类型 | 说明 | 所需材料 |
|----------|------|----------|
| 联系方式变更 | 电话、地址变更 | 身份证明 |
| 受益人变更 | 指定/变更受益人 | 身份证明、关系证明 |
| 缴费方式变更 | 年缴/月缴切换 | 申请表格 |
| 保额变更 | 增额/减额 | 健康告知、核保 |
| 保单贷款 | 保单质押贷款 | 身份证明 |
| 保单复效 | 失效保单恢复 | 健康告知、补交保费 |
| 退保 | 解除保险合同 | 身份证明、保单 |
