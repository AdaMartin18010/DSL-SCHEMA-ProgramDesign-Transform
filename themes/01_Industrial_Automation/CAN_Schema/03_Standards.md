## 1. 标准体系概述

CAN协议Schema标准体系分为三个层次：

1. **国际标准**：ISO等国际组织制定
2. **行业标准**：行业组织制定
3. **厂商标准**：工具厂商制定

### 1.1 标准关系

```text
国际标准（ISO 11898）
    ↓
行业标准（SAE J1939、CANopen等）
    ↓
厂商标准（Vector DBC、CANoe等）
```

## 2. 国际标准

### 2.1 ISO 11898-1:2024

**标准名称**：
Road vehicles - Controller area network (CAN) -
Part 1: Data link layer and physical signalling

**发布信息**：

- **发布日期**：2024年5月24日
- **替代版本**：ISO 11898-1:2015（已撤回）

**核心内容**：

- **数据链路层定义**：
  - 帧结构（标准帧、扩展帧）
  - 仲裁机制
  - 错误检测和处理
  - MAC子层规范

- **三代协议支持**：
  - **CAN CC（Classical CAN）**：经典CAN，8字节数据帧
  - **CAN FD（Flexible Data-rate）**：灵活数据速率，64字节数据帧
  - **CAN XL（Extra Long）**：超长数据帧，2048字节数据帧

- **新增CAN XL支持**：
  - 支持2048字节数据帧
  - 增强数据速率
  - 新的帧格式定义

- **新增CAN FD Light规范**：
  - 简化版CAN FD协议
  - 降低实现复杂度
  - 适用于成本敏感应用

- **帧格式Schema**：
  - SOF（帧起始）
  - 仲裁场
  - 控制场
  - 数据场
  - CRC场
  - ACK场
  - EOF（帧结束）

**Schema体现**：
ISO 11898-1明确定义了CAN帧的位场结构，
这是数据链路层Schema的核心。

**最新版本**：2024版

**参考链接**：
[ISO官网](https://www.iso.org/)

### 2.2 ISO 11898-2:2024

**标准名称**：
Road vehicles - Controller area network (CAN) -
Part 2: High-speed medium access unit

**发布信息**：

- **发布日期**：2024年3月22日
- **替代版本**：ISO 11898-2:2016

**核心内容**：

- **高速CAN物理层定义**
- **电气特性规范**
- **差分电压规范**
- **波特率定义**
- **最高速率提升至20 Mbit/s**（原8 Mbit/s）
- **完整支持CAN XL物理层**

**Schema体现**：
定义了物理层的电气特性Schema。

**最新版本**：2024版

### 2.3 ISO 11898-3:2006

**标准名称**：
Road vehicles - Controller area network (CAN) -
Part 3: Low-speed, fault-tolerant,
medium-dependent interface

**核心内容**：

- **低速容错CAN物理层定义**
- **容错机制**
- **单线CAN支持**

**Schema体现**：
定义了容错CAN的物理层Schema。

**最新版本**：2006版

### 2.4 ISO 11898-4:2004

**标准名称**：
Road vehicles - Controller area network (CAN) -
Part 4: Time-triggered communication

**核心内容**：

- **时间触发CAN（TTCAN）**
- **时间同步机制**
- **确定性通信**

**Schema体现**：
定义了时间触发通信的Schema。

## 3. 行业标准

### 3.1 SAE J1939

**组织**：SAE（美国汽车工程师学会）

**核心标准**：

#### 3.1.1 SAE J1939-71:2025

- **标题**：Vehicle Application Layer
- **发布日期**：2025年2月25日
- **替代版本**：SAE J1939-71:2020
- **内容**：应用层消息和信号定义
- **新增内容**：
  - **J1939DA-2025数字附件**：全新的数字化参数定义附件
  - **StasHH对齐参数**：氢燃料存储标准对齐参数组
  - **新能源参数组扩展**：电动汽车、燃料电池车专用参数
  - **自动驾驶参数组扩展**：ADAS、自动驾驶相关信号定义
- **Schema**：参数组（PGN）和可疑参数编号（SPN）

#### 3.1.2 SAE J1939-73:2024

- **标题**：Application Layer - Diagnostics
- **发布日期**：2024年12月3日
- **替代版本**：SAE J1939-73:2020
- **内容**：诊断消息定义
- **新增内容**：
  - 新能源汽车诊断故障码扩展
  - 自动驾驶系统诊断支持
  - 网络安全诊断功能
- **Schema**：诊断故障码（DTC）定义

**应用领域**：商用车（卡车、客车、工程机械）

**影响**：商用车行业事实标准

**参考链接**：
[SAE官网](https://www.sae.org/)

### 3.2 CANopen

**组织**：CAN in Automation（CiA）

**核心标准**：

#### 3.2.1 CiA 301

- **标题**：CANopen Application Layer and
  Communication Profile
- **内容**：对象字典、PDO、SDO定义
- **Schema**：对象字典Schema

#### 3.2.2 CiA 401-410

- **标题**：I/O Modules Device Profile
- **内容**：I/O模块设备配置
- **Schema**：设备配置Schema

**应用领域**：工业自动化

**影响**：工业自动化CAN协议标准

**参考链接**：
[CiA官网](https://www.can-cia.org/)

### 3.3 DeviceNet

**组织**：ODVA（Open DeviceNet Vendor Association）

**核心内容**：

- **基于CAN的设备网络协议**
- **设备配置Schema**
- **I/O数据交换Schema**

**应用领域**：工业设备网络

**影响**：工业设备网络标准

**参考链接**：
[ODVA官网](https://www.odva.org/)

### 3.4 NMEA 2000

**组织**：NMEA（National Marine Electronics Association）

**核心内容**：

- **船舶电子设备CAN协议**
- **消息定义Schema**
- **设备配置Schema**

**应用领域**：船舶电子设备

## 4. 厂商标准

### 4.1 Vector DBC格式

**厂商**：Vector Informatik GmbH

**标准格式**：

- **DBC文件格式**：
  事实上的CAN应用层Schema标准格式
- **DBC语法**：
  明确定义的文本格式
- **工具支持**：
  CANoe、CANalyzer等工具支持

**Schema特点**：

- 消息定义Schema
- 信号定义Schema
- 节点定义Schema
- 属性定义Schema

**影响**：被广泛采用作为CAN应用层Schema标准

### 4.2 CANdb++格式

**厂商**：Vector Informatik GmbH

**标准格式**：

- **CANdb++数据库格式**：
  二进制格式的CAN数据库
- **与DBC的关系**：
  CANdb++可以导出为DBC格式

## 6. 标准发展趋势

### 6.1 2024-2025年趋势

#### 6.1.1 CAN FD扩展

- **趋势**：CAN FD（Flexible Data-rate）普及
- **影响**：需要新的Schema定义支持64字节数据
- **标准**：ISO 11898-1:2024已全面支持

#### 6.1.2 CAN XL

- **趋势**：CAN XL（下一代CAN协议）**已正式发布**
- **状态**：✅ ISO 11898-1:2024（2024年5月发布）、ISO 11898-2:2024（2024年3月发布）
- **影响**：需要全新的Schema定义支持2048字节数据帧
- **标准**：ISO 11898-1:2024、ISO 11898-2:2024

#### 6.1.3 时间敏感网络（TSN）

- **趋势**：TSN与CAN集成
- **影响**：需要时间同步Schema
- **标准**：IEEE 802.1 TSN标准

#### 6.1.4 2024-2025年新进展

- **2024年3月**：ISO 11898-2:2024发布，物理层支持20 Mbit/s和CAN XL
- **2024年5月**：ISO 11898-1:2024发布，定义CAN XL数据链路层规范
- **2024年12月**：SAE J1939-73:2024发布，新增新能源汽车诊断支持
- **2025年2月**：SAE J1939-71:2025发布，新增氢燃料和自动驾驶参数组

### 6.2 标准化方向

1. **统一性**：推动应用层Schema统一
2. **互操作性**：增强不同标准互操作
3. **可扩展性**：支持新协议扩展（CAN CC/CAN FD/CAN XL三代兼容）
4. **安全性**：加强安全相关Schema定义
5. **新能源适配**：支持电动汽车、氢燃料等新能源车辆通信需求

### 6.3 2025-2026年展望

#### 6.3.1 CAN XL标准化

- **状态**：✅ ISO 11898-1:2024和ISO 11898-2:2024已发布
- **影响**：支持2048字节数据帧，速率可达20 Mbit/s
- **Schema需求**：需要新的消息格式定义
- **下一步**：工具链和应用层协议适配

#### 6.3.2 安全增强

- **趋势**：ISO 14229（UDS）安全扩展
- **影响**：需要安全认证Schema
- **标准**：ISO 14229-2安全扩展

#### 6.3.3 云原生集成

- **趋势**：CAN数据云端处理
- **影响**：需要JSON/YAML Schema转换
- **标准**：基于OpenAPI/AsyncAPI转换
