# 贡献指南

## 欢迎贡献

感谢您对DSL Schema Themes项目的关注！我们欢迎各种形式的贡献。

## 贡献方式

### 1. 提交问题 (Issues)

发现问题或有建议？请提交Issue：
- 描述清楚问题
- 提供复现步骤
- 说明预期行为

### 2. 改进文档

- 修正错误
- 补充示例
- 翻译内容

### 3. 添加案例

分享您的行业实践：
- 案例背景
- Schema设计
- 实现代码
- 经验总结

### 4. 开发工具

- 新增验证器
- 开发转换器
- 创建可视化工具

## 贡献流程

1. Fork项目
2. 创建分支 (`git checkout -b feature/YourFeature`)
3. 提交更改 (`git commit -m 'Add some feature'`)
4. 推送分支 (`git push origin feature/YourFeature`)
5. 创建Pull Request

## 文档规范

### Markdown格式
- 使用标准Markdown语法
- 标题使用#格式
- 代码块标注语言

### 代码规范
- Python遵循PEP 8
- 添加类型注解
- 编写docstring

## 审核流程

- 所有贡献需经过审核
- 审核标准：准确性、完整性、实用性
- 通常在3个工作日内回复
