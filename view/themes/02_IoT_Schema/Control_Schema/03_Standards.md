## 1. 标准体系概述

IoT控制Schema标准体系分为四个层次：

1. **国际标准**：IEC等国际组织制定
2. **国家标准**：各国标准化组织制定
3. **行业标准**：行业组织制定
4. **厂商标准**：设备厂商和平台厂商制定

### 1.1 标准关系

```text
国际标准（IEC）
    ↓
国家标准（GB/T）
    ↓
行业标准（IO-Link、OPC UA、MQTT）
    ↓
厂商标准（设备厂商）
```

## 2. 国际标准

### 2.1 IEC 61131-3

**标准名称**：
Programmable controllers - Part 3: Programming languages

**核心内容**：

- **控制语言**：ST、LD、FBD、SFC、IL
- **任务调度**：任务定义、优先级、周期
- **功能块**：标准功能块、自定义功能块

**Schema体现**：
IEC 61131-3定义了PLC控制逻辑的Schema。

**最新版本**：IEC 61131-3:2013

**参考链接**：
[IEC官网](https://webstore.iec.ch/)

### 2.2 IEC 61499

**标准名称**：
Function blocks

**核心内容**：

- **功能块模型**：事件驱动功能块
- **分布式控制**：分布式控制系统
- **执行模型**：执行控制图

**Schema体现**：
IEC 61499定义了分布式控制系统的Schema。

**最新版本**：IEC 61499:2012

**参考链接**：
[IEC官网](https://webstore.iec.ch/)

## 3. 国家标准

### 3.1 GB/T 34068-2017

**标准名称**：
物联网总体技术 智能传感器接口规范

**核心内容**：

- **第7章**：控制与配置
- **采样控制**：采样模式、频率、触发条件
- **参数配置**：参数定义、验证、持久化
- **事件管理**：事件定义、触发、处理

**Schema体现**：
GB/T 34068-2017定义了IoT设备控制的Schema。

**状态**：现行有效

### 3.2 GB/T 33008.1-2016

**标准名称**：
可编程控制器 第1部分：通用信息

**核心内容**：

- **控制逻辑**：程序组织单元
- **任务调度**：任务定义和调度
- **功能块**：功能块定义和使用

**Schema体现**：
GB/T 33008.1-2016定义了PLC控制的Schema。

**状态**：现行有效

## 4. 行业标准

### 4.1 IO-Link

**组织**：IO-Link Consortium

**标准名称**：
IO-Link Specification

**核心内容**：

- **参数配置**：IODD文件定义参数
- **事件管理**：事件定义和处理
- **诊断功能**：设备诊断和状态

**Schema体现**：
IO-Link IODD文件定义了设备控制的Schema。

**最新版本**：IO-Link Specification V1.1.3

**参考链接**：
[IO-Link官网](https://www.io-link.com/)

### 4.2 OPC UA

**组织**：OPC Foundation

**标准名称**：
OPC Unified Architecture

**核心内容**：

- **方法调用**：设备方法调用
- **数据订阅**：数据变化订阅
- **事件通知**：事件通知机制

**Schema体现**：
OPC UA信息模型定义了设备控制的Schema。

**最新版本**：OPC UA 1.05

**参考链接**：
[OPC Foundation](https://opcfoundation.org/)

### 4.3 MQTT

**组织**：OASIS

**标准名称**：
MQTT - Message Queuing Telemetry Transport

**核心内容**：

- **命令下发**：通过MQTT主题下发命令
- **状态上报**：通过MQTT主题上报状态
- **QoS保证**：消息质量保证

**Schema体现**：
MQTT主题结构和消息格式定义了设备控制的Schema。

**最新版本**：MQTT 5.0

**参考链接**：
[MQTT官网](https://mqtt.org/)
