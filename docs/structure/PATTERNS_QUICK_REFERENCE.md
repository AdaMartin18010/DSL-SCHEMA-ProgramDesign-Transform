## 3. 架构模式速查

### 3.1 分层架构模式（3个）

| 模式 | 适用场景 | 在DSL Schema转换中的应用 |
|------|---------|------------------------|
| **三层架构** | 传统Web应用、企业级应用、简单系统 | 表示层（API接口）、业务层（转换引擎）、数据层（PostgreSQL） |
| **四层架构** | 领域驱动设计（DDD）、复杂业务系统 | 表示层、应用层（用例编排）、领域层（领域模型）、基础设施层 |
| **五层架构** | 大型企业系统、多数据源系统 | 表示层、应用层、领域层、基础设施层、数据层（独立数据访问） |

| 模式 | 适用场景 | 在DSL Schema转换中的应用 |
|------|---------|------------------------|
| **API网关模式** | 微服务架构、统一API管理 | 路由、认证、限流、监控 |
| **服务发现模式** | 服务动态部署、容器化部署 | 服务注册、发现、负载均衡 |
| **配置中心模式** | 集中配置管理、多环境配置 | 转换规则配置、验证规则配置 |
| **熔断器模式** | 服务容错、防止服务雪崩 | 转换服务容错、外部API调用容错 |

### 3.3 事件驱动架构模式（3个）

| 模式 | 适用场景 | 在DSL Schema转换中的应用 |
|------|---------|------------------------|
| **发布-订阅模式** | 事件驱动架构、解耦组件 | Schema变更通知、转换完成通知 |
| **事件溯源模式** | 事件历史、审计需求 | 转换历史记录、状态重建 |
| **CQRS模式** | 读写分离、性能优化 | 写模型（Schema创建/更新）、读模型（Schema查询） |

### 3.4 数据架构模式（3个）

|------|---------|------------------------|
| **数据仓库模式** | 数据仓库建设、数据分析 | Schema元数据仓库、转换历史仓库 |
| **数据湖模式** | 大数据存储、原始数据保留 | 原始Schema存储、转换结果存储 |
| **数据网格模式** | 大型企业数据管理、分布式数据架构 | 领域数据产品、数据目录、数据治理 |
