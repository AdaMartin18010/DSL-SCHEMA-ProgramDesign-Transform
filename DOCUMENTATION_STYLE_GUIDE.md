# DSL Schema 文档格式规范

## 📋 格式规范

### 1. 目录结构

所有文档必须在开头包含完整的目录结构，使用以下格式：

```markdown
## 📑 目录

- [文档标题](#文档标题)
  - [1. 一级标题](#1-一级标题)
    - [1.1 二级标题](#11-二级标题)
      - [1.1.1 三级标题](#111-三级标题)
```

### 2. 标题编号

- **一级标题**：使用 `## 1. 标题` 格式（注意：`##` 后有一个空格，然后是数字和点）
- **二级标题**：使用 `### 1.1 标题` 格式
- **三级标题**：使用 `#### 1.1.1 标题` 格式
- **四级标题**：使用 `##### 1.1.1.1 标题` 格式

### 3. 行长度控制

为了适应移动端阅读，每行字符数应控制在 **50-75字符** 之间。

**示例**：

```markdown
<!-- ❌ 错误：行太长 -->
这是一个非常长的句子，超过了75个字符的限制，在移动设备上阅读会很困难。

<!-- ✅ 正确：行长度适中 -->
这是一个长度适中的句子，
在移动设备上阅读会很舒适。
```

### 4. 表格格式

表格应使用简洁的格式，避免过宽的列：

```markdown
| 列1 | 列2 | 列3 |
|-----|-----|-----|
| 数据1 | 数据2 | 数据3 |
```

### 5. 代码块格式

代码块应包含语言标识：

```markdown
    ```yaml
    # YAML示例
    key: value
    ```

    ```python
    # Python示例
    def example():
        pass
    ```

```

### 6. 数学公式格式

数学公式使用 LaTeX 格式：

```markdown
行内公式：$H(s) = -\sum_{i=1}^{n} p(e_i) \log_2 p(e_i)$

块级公式：
$$
H(s) = -\sum_{i=1}^{n} p(e_i) \log_2 p(e_i)
$$
```

### 7. 引用格式

使用标准的 Markdown 引用格式：

```markdown
> 这是一个引用块
> 可以包含多行内容
```

### 8. 链接格式

使用描述性链接文本：

```markdown
[链接文本](URL)
```

### 9. 列表格式

- 无序列表使用 `-` 或 `*`
- 有序列表使用数字编号
- 嵌套列表使用2个空格缩进

### 10. 分隔线

使用 `---` 作为章节分隔线。
